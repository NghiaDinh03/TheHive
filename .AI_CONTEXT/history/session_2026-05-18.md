# Session 2026-05-18: Nâng cấp Bảo mật 2FA, Dynamic Session & Step-Up Auth

## Mục tiêu
Hardening NCS Fusion Center: triển khai 2FA bắt buộc, Session Management động (4/8/10h), và Step-Up Authentication cho tác vụ Admin nhạy cảm.

## 1. Database Migration (`000041_auth_enhancements`)
- Đổi tên tổ chức `admin` → `NCS`
- Xóa user rác `ncs.fushion_admin`
- Thêm cột `session_duration_hours` (INT, default 4) vào bảng `users`
- Thêm cột `force_2fa` (BOOLEAN, default false) vào bảng `users`

## 2. Dynamic Session Management
- **Backend (`auth.go`)**: Hàm `completeLogin` đọc `session_duration_hours` của user → set `exp` JWT token theo mốc do user chọn (4, 8, 10 tiếng thay vì hardcode).
- **Backend (`auth.go` → `UpdateMe`)**: Hỗ trợ PATCH `session_duration_hours` vào profile cá nhân.
- **Frontend (`personal-settings/page.tsx`)**: Thêm Dropdown chọn Session Duration (4, 8, 10 tiếng) trong trang Cá nhân, tự động lưu khi user nhấn Save.

## 3. Force 2FA (Ép buộc Bảo mật 2 Bước)
- **Backend (`totp.go`)**: `/totp/setup` luôn tạo mã bí mật mới mỗi lần gọi (tránh QR reuse). Nếu user đã bật `totp_enabled=true`, API trả 400 reject.
- **Backend (`auth.go`)**: Login kiểm tra `totp_enabled` → yêu cầu `X-TOTP-Code` header. Nếu thiếu, trả `totp_required`.
- **Frontend (`Force2FAOverlay.tsx`)**: Component mức Global trong `layout.tsx`. Khi `force_2fa=true` nhưng `totp_enabled=false`, overlay full-screen chặn mọi thao tác, chỉ cho phép scan QR + xác nhận mã 6 số.
- **Admin UI (`admin/page.tsx`)**: Thêm checkbox "Bắt buộc bật 2FA (Bảo mật)" trong form Edit User. Hiển thị badge "Bắt buộc 2FA" trong danh sách user.

## 4. Step-Up Authentication (Xác thực tác vụ nhạy cảm Admin)
- **Backend Middleware (`middleware.go` → `RequireStepUp2FA`)**: Kiểm tra header `X-TOTP-Code` trên mỗi request mutate. Nếu thiếu hoặc sai → reject 401.
- **Backend Routes (`routes_auth.go`)**: Gắn `RequireStepUp2FA` vào tất cả route POST/PATCH/DELETE của `/admin/users`, `/admin/organisations`, `/admin/profiles`.
- **Frontend (`admin/page.tsx` → `stepUpFetch`)**: Wrapper bắt lỗi 401 `missing X-TOTP-Code` → tự động bật Popup Modal yêu cầu admin nhập mã 2FA 6 số → retry request với header `X-TOTP-Code`.
- **Brute-force Protection**: Gắn `RateLimiter` vào `/totp/verify`, `/totp/disable` (10 request/5 phút, lockout 15 phút).

## 5. Export & Cleanup
- Export `verifyTOTPCode` → `VerifyTOTPCode` (public) trong `totp.go` để middleware `RequireStepUp2FA` trong package `server` có thể gọi được.
- Cập nhật tất cả references trong `auth.go` và `totp.go`.
- Thêm `force_2fa` vào struct `adminUserSummary`, `adminUserUpdateRequest` và các SQL query tương ứng trong `admin.go`.

## 8. Lưu ý Bảo mật
- Mã QR Code TOTP luôn được tạo mới mỗi lần `/totp/setup` → không thể reuse QR cũ
- Step-Up 2FA bảo vệ cả API route level → không bypass được dù có JWT hợp lệ
- Rate Limiter: 10 req/5 phút cho TOTP verify/disable, lockout 15 phút
- Admin có thể toggle `force_2fa` cho từng user riêng biệt
- Session duration tùy chỉnh per-user (4/8/10h), default 4h

  - Đã thêm lệnh `url.PathUnescape` và import `"net/url"` vào toàn bộ các handler trong `admin.go` để decode tham số `c.Param("login")`. Lỗi "user not found" khi xóa/khóa tài khoản đã được xử lý triệt để.
